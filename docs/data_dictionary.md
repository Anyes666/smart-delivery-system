# 系统数据字典

> 对标标准: GB/T 22263-2008 物流信息系统技术规范

## 一、系统标识

| 属性 | 值 |
|------|-----|
| 系统名称 | 基于真实路况与在线自适应预测的智能配送优化系统 |
| 系统版本 | 2.0 |
| 数据字典版本 | 1.0 |
| 编制日期 | 2026-05-18 |
| 对标标准 | GB/T 22263-2008, GB/T 37378-2019, GB/T 18354-2021, GB/T 28583-2012, JT/T 1325-2020 |

---

## 二、输入数据格式

### 2.1 配送点 CSV 文件

| 字段名 | 类型 | 单位 | 取值范围 | 是否必填 | 说明 |
|--------|------|------|---------|---------|------|
| 经度 | float | 度(°) | 73.0–135.0 | 是 | WGS84 坐标系, GCJ-02 需标注 |
| 纬度 | float | 度(°) | 18.0–54.0 | 是 | WGS84 坐标系 |
| 需求量 | float | 件/吨 | 0–100000 | 是 | 配送点的物品数量/重量 |
| 时间窗开始 | int | 秒(当日) | 0–86400 | 否 | 最早到达时间, 默认 0 |
| 时间窗结束 | int | 秒(当日) | 0–86400 | 否 | 最晚到达时间, 默认 86400 |
| 优先级 | int | — | 0–10 | 否 | 0=配送中心, 1-10=客户优先级, 默认 1 |
| 点编号 | int | — | ≥0 | 是 | 0号点为配送中心 (GB/T 37378 设施编码) |

### 2.2 训练数据 (JSONL)

| 字段名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| origin_lon | float | 度(°) | 起点经度 |
| origin_lat | float | 度(°) | 起点纬度 |
| dest_lon | float | 度(°) | 终点经度 |
| dest_lat | float | 度(°) | 终点纬度 |
| dist_km | float | km | 实际道路距离 |
| duration_seconds | float | s | 实际行程时间 |
| hour | int | — | 出发时间 (小时, 0-23) |
| day_of_week | int | — | 出发日期 (0=周一) |
| timestamp | float | s | Unix 时间戳 |
| city | str | — | 城市标识 |
| real_time | bool | — | 是否真实时间采集 |

## 三、输出数据格式

### 3.1 路线结果 (JSON, 对标 GB/T 28583-2012)

| 字段名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| 运单号 | str | — | 格式: SDS-YYYYMMDD-V{序号} |
| 路线编号 | str | — | 格式: R-{序号:04d} |
| 车辆编号 | str | — | 格式: V-{序号:04d} |
| 车型代码 | str | — | JT/T 1325 代码 (V-MINI/V-LIGHT/V-MEDIUM/V-HEAVY) |
| 配送中心点编号 | int | — | 发车点索引 |
| 途经点序列 | list[int] | — | 配送点访问顺序 |
| 停靠站数 | int | 个 | 配送停靠站点数 |
| 总行驶里程 | float | km | 路线总距离 |
| 总装载量 | float | 件/吨 | 路线总需求 |
| 装载率 | float | % | 装载量/容量 × 100 |

### 3.2 对比实验结果 (CSV)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 方法 | str | 方法名称 |
| 距离均值km | float | 5次实验平均总距离 |
| 距离标准差km | float | 标准差 |
| 车辆数 | int | 实际使用车辆数 |
| 利用率% | float | 车辆利用率 |
| 求解用时s | float | 求解耗时 |
| 溢出点 | int | 未分配点数 |

## 四、车型定义 (JT/T 1325-2020)

| 车型名称 | 代码 | 最大载重(t) | 典型容量 | 车长范围(m) | 能源类型 | 适用场景 |
|---------|------|-----------|---------|-----------|---------|---------|
| 微型封闭货车 | V-MINI | ≤1.8 | 15 | 2.5–4.5 | 电动/汽油 | 社区配送、快递末端 |
| 轻型封闭货车 | V-LIGHT | 1.8–4.5 | 30 | 4.5–6.0 | 电动/汽油/柴油 | 城市配送、商超冷链 |
| 中型厢式货车 | V-MEDIUM | 4.5–12.0 | 60 | 6.0–9.0 | 柴油/电动 | 区域配送、仓储转运 |
| 重型厢式货车 | V-HEAVY | 12.0–25.0 | 120 | 9.0–12.0 | 柴油/LNG | 干线运输、大宗物资 |

## 五、接口规范 (GB/T 22263-2008)

### 5.1 输入接口

```
POST /api/v1/optimize
Content-Type: application/json

{
  "depot": {"lon": 116.40, "lat": 39.90, "facility_code": "F-DC1101010001"},
  "customers": [
    {"lon": 116.41, "lat": 39.91, "demand": 10, "time_window": [28800, 36000], "priority": 2}
  ],
  "fleet": {"max_vehicles": 5, "vehicle_types": ["V-LIGHT", "V-MEDIUM"]},
  "options": {"algorithm": "ortools", "time_limit": 30}
}
```

### 5.2 输出接口

```
HTTP 200 OK
Content-Type: application/json

{
  "status": "success",
  "data_standard": "GB/T 28583-2012",
  "solution": {
    "total_distance_km": 45.2,
    "total_time_s": 7200,
    "vehicles_used": 3,
    "routes": [...]
  }
}
```
